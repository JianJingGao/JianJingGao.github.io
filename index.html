2.1 常量

——–*nullptr

·其出现的目的为了替代NULL，某种意义上，传统C++会把NULL、0看作同一种东西，取决编译器如何定义NULL，有些编译器会定义为((void*)0)，有的则直接会将其定义为0。

C++不允许直接void*隐式转换其他类型

因为若有:

Char* ch=NULL;

没有隐式转换的cpp将NULL定义为0，这依然会产生新问题，重载特性会混乱

Void foo(char*);

Void foo(int);

那么foo(NULL),这个语句则会调用foo(int);

因为这个问题所以出现了nullptr关键字，专门区分空指针、0，而nullptr的类型为nullptr_t，能够隐式转换为任何指针或者成员指针的类型，也可进行相等或不等的比较。

* Nullptr!=0

* decltype用于类型推导

* std::is_same用于比较两个类型是否相等

——–*constexpr

C++11提供了constexpr让用户显示的声明函数或者对象构造函数再编译器会成为常量表达式，这个关键字明确告诉编译器应验证fun在编译器就应该是一个常量表达式。

2.5 模板

​ 模板能将一切在编译期处理的问题丢到编译器处理，仅在运行时处理最核心的动态服务，从而大幅优化运行期性能。因此被视为C++的黑魔法之一

外部模板

​ 在传统C++中，模板在使用时才会被编译器实例化。换句话，只要在每个编译单元中编译的代码遇到了被完整定义的模板，都会被实例化。从而产生重复实例化而导致编译的时间变长。而且我们无法做到通知编译器不要触发模板实例化。

​ 为此在C++11引入了外部模板，扩充了原来的强制编译器在特定位置实例化模板的语法，使我们能显示的通知编译器何时进行模板的实例化：

​ template class std::vector; //强制实例化

​ extern template class std::vector; //不在该当前编译文件中实例化模板

尖括号“>”

在传统C++编译器中，>>一律被当作右移运算符处理，但实际我们很容易就写出了嵌套模板的代码：

​ std::vector<std::vector> matrix;

​ 这在传统C++编译器下是不能被编译的，而C++11开始，连续的右尖括号将变得合法，并且能够顺利通过编译。甚至下面的写法也可以：

变长参数模板

模板一直是C++所独有的黑魔法之一。在C++11之前，无论类模板还是函数模板，都智能按其指定样子，接收一组固定数量的模板参数；而在C++11加入了新的表示方法，允许任意个数，任意类别的模板参数，同时也不需要在定义时将参数个人固定

​ template<typename… Ts> class Magic;

模板类Magic的对象能够不受限制的typename作为模板的形参，例如

​ class Magic, std::vector,std::map<std::string,std::vector>> darkMagic;

既然是任意形式，所以个数为0的模板参数也是可以的：class Magic<> nothing;

template<typename Require,typename …Args>class Magic;

变长参数模板也能被直接调整到模板函数上。传统C中的printf函数，虽然也能达成补丁个数的形参调用，但并非类型安全。C++11除了可以定义类别安全的的变长参数函数外，还可以使类似printf的函数能自然地处理非自带类别的对象。除了在模板函数中能使用…表示不定长模板参数外，函数参数也使同样的表示法代表不定长参数，这也就为我们简单编写变长参数模板提供了便捷的手段：

一、环境安装与配置

​ 今天开始学习Golang语言，下面记录每一步学习的节点总结。学习一门编程语言，最基础的就是要搭建其运行环境和熟悉基本语法了，搭建Golang的运行环境则需要先下载相应的二进制文件。

​ Golang是一个由Google开发的开源的编译型的静态语言，他的主要关注点是使得高可用性和可扩展性的web应用开发变得简易。

​ Go语言的优点有：

  • ​ 并发是语言的一部分，编写多线程程序会非常容易。主要倚靠Goroutine和channels机制实现
  • ​ Golang是一种编译型语言。源代码会被编译为二进制机器码，而解释型语言没有这个过程
  • ​ 支持静态链接，所以代码可以静态链接为一个大的二进制文件

安装

1.下载语言包及IDE

​ Golang支持三个主流平台，其有个特性是可以跨平台编译，即A平台编译B平台运行。你可以在 https://golang.org/dl/中下载相应平台的二进制文件。

​ IDE：集成开发环境，使用顺手的IDE可以有效帮助我们提升开发效率，Golang开发中常用的IDE有VS Code、Jetbrains Goland等，目前我选择的是第二个，因为…..顺手啊~前者免费，后者收费的，但是后者易用性和界面布局都让我很喜欢。VS Code还需要另外安装插件。目前我是使用教育认证，取得了一年期的license使用。

​ Jetbrains Goland：点击下载

2.配置环境

Mac OS

在 https://golang.org/dl/ 下载安装程序。双击开始安装并且遵循安装提示，会将 Golang 安装到 /usr/local/go 目录下，同时 /usr/local/go/bin 文件夹也会被添加到 PATH 环境变量中。

Windows

在 https://golang.org/dl/ 下载 MSI 安装程序。双击开始安装并且遵循安装提示，会将 Golang 安装到 C:\Go 目录下，同时 c:\Go\bin 目录也会被添加到你的 PATH 环境变量中。

Linux

在 https://golang.org/dl/ 下载 tar 文件，并解压到 /usr/local。

请添加 /usr/local/go/bin 到 PATH 环境变量中。Go 就已经成功安装在 Linux 上了。

3.配置IDE
